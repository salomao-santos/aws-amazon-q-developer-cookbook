# Documentation Recipes

This directory contains tested prompts for creating and optimizing documentation with Amazon Q Developer.

## Available Recipes

1. [API Documentation](./api-documentation.md) - Create comprehensive API documentation
2. [Code Comments](./code-comments.md) - Generate helpful inline comments
3. [README Files](./readme.md) - Create informative README files
4. [Architecture Documentation](./architecture.md) - Document system architecture
5. [User Guides](./user-guides.md) - Create end-user documentation

## Usage

Each recipe includes:
- **Prompt Template**: Ready-to-use prompt for Amazon Q Developer
- **Documentation Examples**: Sample outputs
- **Best Practices**: Tips for clear documentation
- **Templates**: Reusable documentation structures
