# Security Recipes

This directory contains tested prompts for integrating security best practices with Amazon Q Developer.

## Available Recipes

1. [Input Validation](./input-validation.md) - Validate and sanitize user input
2. [Authentication & Authorization](./auth.md) - Implement secure authentication
3. [SQL Injection Prevention](./sql-injection.md) - Prevent SQL injection attacks
4. [XSS Prevention](./xss-prevention.md) - Prevent cross-site scripting
5. [Secrets Management](./secrets-management.md) - Securely handle sensitive data
6. [API Security](./api-security.md) - Secure REST and GraphQL APIs
7. [AWS Security](./aws-security.md) - AWS-specific security best practices

## Usage

Each recipe includes:
- **Prompt Template**: Ready-to-use prompt for Amazon Q Developer
- **Security Examples**: Before/after code examples
- **Best Practices**: Security guidelines and standards
- **Common Vulnerabilities**: How to identify and fix them
